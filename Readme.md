# ChatGPT メモリバックアップ用プロンプト

## 概要
プロンプトを ChatGPT に入力すると、メモリデータ(Model Set Context)を可能な範囲でバックアップし、JSONファイルまたはMarkdownファイルとしてダウンロードできます。
下の“プロンプト”から形式を選んでください（JSON / Markdown）。

## プロンプト
リンク先で Raw を押して全文コピペしてね。
- あとで編集・解析したい？ → [JSONのみプロンプト](./prompts/json_only.md)
- 眺めたい・保存したい？ →[Markdownのみプロンプト](./prompts/markdown_only.md)

### 🛠 使い方
負荷が高い処理が連続するため、新規チャットを立ち上げて最新の**Thinkingモデルの拡張思考**を指定してから実行することを推奨します。
チャット入力欄にプロンプトを貼り付けてChatGPTに送信してください。

### どっちのプロンプトを使う？
- **JSON版（おすすめ: エンジニア向け）**  
  検証（sha256）や重複チェックができて、後から加工・統合しやすい。  
  「正確に保存したい」「将来また機械で扱いたい」人向け。

- **Markdown版（おすすめ: 読む用）**  
  そのまま読める“思い出アルバム”形式。  
  「とにかく読みやすく残したい」「手元で眺めたい」人向け。

迷ったら：まず **Markdown**（読みやすい） → きっちり残したくなったら **JSON**。

## 注意点
- メモリの仕様が変更されると、動作しなくなる可能性があります。
- 生成されたデータが完璧にメモリを再現する保証はありません。
- **バックアップファイルは個人情報を含む可能性があるので、保管場所に注意してね。**

## うまくいかないとき（任意）
- 新規チャットで実行（推奨）
- 返答が途切れる/重い場合は、**一時的に**カスタム指示を短くする（または空にする）と安定することがあります
- セッション切れでダウンロードできない場合は、同じプロンプトで再実行してください（内容は再生成されます）

## バックアップ後のおすすめ
- **Markdown版**は Notion / Obsidian に貼ると「思い出アルバム」になって幸せになりがち
- **JSON版**は「厳密な保存・検証・後から加工」向け（必要ならMarkdownに変換して読むのもOK）

### Gitで管理したい人へ（任意）
- ローカルの **プライベートリポジトリ** に置くと、差分が追えて便利（UKMの変化を見るのも面白い）
- ただし **個人情報が含まれる可能性**があるので、公開リポジトリに誤ってpushしないよう注意
- `.gitignore` に `memory_*.json` / `memory_*.md` を入れておくと安心
